# asset-library Specification

## Purpose

以 Blob 形式管理本地图片素材库,分离图字节与生成元数据并以稳定 id 相互引用,支持预览与在后续生成中复用素材。

## Requirements

### Requirement: 素材库存储
系统 SHALL 将每张图片以 Blob 形式存入本地素材库,图字节与生成元数据分离,并以稳定 id 相互引用;素材默认不过期。

#### Scenario: 图片以 Blob 落库
- **WHEN** 一次生成产出图片
- **THEN** 每张图片作为独立 Blob 存储并分配唯一 id,生成记录以该 id 引用,不内联复制图字节

#### Scenario: 一图多用不复制
- **WHEN** 同一张图既是某次生成的输出、又被另一次生成用作参考图
- **THEN** 系统仅存储一份图字节,由多条记录以 id 引用

### Requirement: 素材预览与复用
系统 SHALL 展示素材库中的图片供预览,并允许将任一素材选作后续生成的参考图。

#### Scenario: 预览素材
- **WHEN** 用户打开素材库
- **THEN** 系统展示已存图片的缩略预览

#### Scenario: 选作参考图
- **WHEN** 用户在 chat 协议下从素材库选择一张图作为参考图
- **THEN** 该图被加入当前生成的参考图集合

### Requirement: 素材收藏
系统 SHALL 允许用户将素材标记为收藏或取消收藏,收藏状态本地持久化,并随整库导出一并保留。

#### Scenario: 标记收藏
- **WHEN** 用户点击某素材的收藏
- **THEN** 系统持久化其收藏状态,再次打开仍保持

#### Scenario: 按收藏筛选
- **WHEN** 用户开启"仅看收藏"
- **THEN** 素材库仅显示已收藏的素材

### Requirement: 引用感知的连带删除
删除生成记录或会话时,系统 SHALL 仅在产出图既未被收藏、又未被任何删除后仍存活的生成记录当作参考图引用时,才连带删除该产出图;满足任一保留条件即保留。参考图 `refImageIds` SHALL NOT 因删除生成/会话而被连带删除。

#### Scenario: 未收藏且无引用则连带删
- **WHEN** 删除某生成,其产出图未被收藏且无其他存活记录将其用作参考图
- **THEN** 系统连带删除该产出图的 Blob

#### Scenario: 已收藏则保留
- **WHEN** 删除某生成,其产出图已被收藏
- **THEN** 系统保留该产出图,仅删除生成记录

#### Scenario: 仍被引用则保留
- **WHEN** 删除某生成,其产出图仍被另一条存活记录用作参考图
- **THEN** 系统保留该产出图,避免其他会话出现裂图

#### Scenario: 参考图不被连带删
- **WHEN** 删除某生成,该生成附带了用户选择的参考图
- **THEN** 系统不删除这些参考图素材

### Requirement: 清空全部素材与记录
系统 SHALL 提供"清空全部"入口,删除所有生成记录与所有素材,但 SHALL 保留接口预设与 API Key;该操作为不可逆破坏性操作,SHALL 在执行前请求二次确认。

#### Scenario: 确认后清空
- **WHEN** 用户在存储与备份中选择清空全部并确认
- **THEN** 系统删除全部生成记录与素材,保留接口预设与 Key,界面回到空状态

#### Scenario: 取消清空
- **WHEN** 用户在确认弹窗中取消
- **THEN** 系统不删除任何数据
