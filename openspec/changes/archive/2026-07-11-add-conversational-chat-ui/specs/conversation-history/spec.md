## ADDED Requirements

### Requirement: 会话持久化
系统 SHALL 将每次"新建创作"作为一段独立会话持久保存;会话内的生成记录不因新建会话而丢失或不可达。

#### Scenario: 新建会话不丢旧记录
- **WHEN** 用户在有历史生成的情况下点击"新建创作"
- **THEN** 系统开启一段空白会话,且此前会话及其生成记录仍完整保存、可再次访问

#### Scenario: 会话标题派生自首个 prompt
- **WHEN** 一段会话产生了第一条生成
- **THEN** 系统以该 prompt 作为会话标题(过长则截断)

### Requirement: 会话导航按日期分组
系统 SHALL 在左侧导航列出历史会话,按日期分组:今天、昨天、本月更早(按天),以及更早月份(按月)。

#### Scenario: 分组展示
- **WHEN** 存在跨越多天与多月的会话
- **THEN** 系统将其归入"今天""昨天""本月更早""更早月份"等分组展示,组内按时间倒序

#### Scenario: 切换会话
- **WHEN** 用户点击导航中的某段历史会话
- **THEN** 系统将画布切换为该会话,展示其全部生成记录

#### Scenario: 当前会话高亮
- **WHEN** 某段会话为当前活动会话
- **THEN** 导航中该项以选中态高亮
