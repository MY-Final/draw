# Proposal

## Problem

纯前端应用,所有数据(接口配置、Prompt、生成记录、图片等)均存储在浏览器 IndexedDB 中。

用户面临以下风险:
- 清除浏览器站点数据
- 更换浏览器或电脑
- 浏览器卸载
- 无痕模式退出

以上操作均可能导致数据永久丢失。目前虽已有整库 zip 导出/导入功能,但入口较深、无主动提醒、缺乏容量可视化,用户容易忽略备份。

## Solution

提供完整的数据保护能力:

1. **存储概览** — 在"存储与备份"面板中展示已用容量、业务图片量、浏览器剩余空间,附健康色标(🟢🟡🔴)
2. **一键导出备份** — 复用现有 zip 导出逻辑,补充 workspace 和 promptLibrary 的导出
3. **一键导入恢复** — 复用现有 zip 导入逻辑,恢复后展示统计结果
4. **备份提醒** — 容量每增加 100MB 触发一次弹窗,30 天冷却期,三个按钮(立即备份/稍后提醒/30天不提醒)
5. **持久化存储** — 检测 `navigator.storage.persisted()`,引导用户启用 `persist()`
6. **UI 重做** — 将现有的多 section 布局改为统一的"数据保护"面板

## 不做的

- 100 张图片触发的备选条件
- 7 天连续使用触发的备选条件
- 存储细分(图片/数据库/缓存无法精确拆分)
